# 详细阅读总结模板与撰写规范

本模板用于「标准工作流」第 7 步：作答完成后，把详细阅读总结落盘到 run 目录根部。它与聊天内回复的简版模板（概要 / 亮点 / 章节大纲）分工不同——简版求快，本模板求全、可留存、可复查。

## 一、文件命名与存放

- 文件名固定为 `【阅读总结】<标题>.md`，保存在 run 目录根部，与 `【文字稿】<标题>.docx`、`【关键帧】<标题>.pdf` 并列。
- `<标题>` 取自 manifest 的 title 并净化：去掉半角 `/\:*?"<>|` 与所有控制字符、首尾 trim、最长 60 字符（超限截断）。
- 三个文件统一以【】前缀命名，是为了在文件管理器里排序相邻、一眼可辨；不要改成其他前缀或省略书名号。

## 二、撰写规范

1. 时间戳：所有关键结论、要点、引用都必须带 `t=MM:SS`（视频超过 1 小时用 `t=HH:MM:SS`）；亚秒级快速操作可补充 `actual_t=192.250s` 或毫秒时码。
2. 证据边界：内容必须基于 transcript 原文与抽帧画面，两者都没有的信息不得编造、不得脑补；证据不足就明说「转写与抽帧均未覆盖」。
3. 引文修正：讲师口语、口误与 AI 字幕 / whisper 的同音错字按语境修正后再引用，不照抄明显错误；修正幅度较大时用「（大意）」标注。
4. 画面印证：涉及演示、板书、界面操作的要点，先对照 `frames/frames.json` 的帧（优先 `actual_t`）确认画面内容再落笔。
5. 超长视频（1 小时以上）：用 map-reduce——按 10～15 分钟一段分批读 transcript 做分段笔记，再汇总成各章节；不要一次性通读后凭印象写。
6. 章节划分：按内容逻辑划分（通常 4～10 章），「内容脉络」表必须覆盖全片、时间首尾相接，与「逐章详解」的小节一一对应。
7. 篇幅与语言：详略跟随信息量，不凑字数；语言与视频及用户语言一致，默认中文。

「视频信息」表的字段来源：`标题`、`来源链接`、`时长` 取 `manifest.json` 的 `title`、`input`、`duration`；`处理日期` 取 `created_at`；`转写方式` 取 `transcript.source`（附 `model`）；`抽帧数量` 取 `frames/frames.json` 的帧数（含补帧 pass，可与 `frames.count` 对照）。

## 三、模板本体

复制以下骨架填写（圆括号为填写说明，成稿时替换掉）：

```markdown
# 【阅读总结】《标题》

## 视频信息

| 项目 | 内容 |
| ---- | ---- |
| 标题 | （原始视频标题） |
| 来源链接 | （URL 或本地文件路径） |
| 时长 | （如 1:41:59） |
| 处理日期 | （YYYY-MM-DD） |
| 转写方式 | （如 faster-whisper / small） |
| 抽帧数量 | （如 100 帧） |

## 一句话总结

（一句话概括全片核心内容，不超过 50 字。）

## 内容脉络

| 起止时间 | 章节 | 内容梗概 |
| -------- | ---- | -------- |
| 00:00–12:30 | （章节名） | （一两句话梗概） |
| … | … | … |

（表格覆盖全片、时间首尾相接。）

## 逐章详解

### 第一章 （章节名）（t=00:00–12:30）

**核心论点**：（1～2 句概括本章在讲什么、结论是什么。）

**要点**：

- （要点 1，带时间戳，如 t=02:15）
- （要点 2，带时间戳）
- （要点 3，带时间戳）
- …（每章至少 3～5 条）

**关键例子/演示**：（本章最具代表性的例子、演示或画面，注明时间戳；抽帧画面印证过的可附一句画面描述。本章没有突出示例则写「本章无突出示例」。）

**本章小结**：（1～2 句收束，可承上启下。）

### 第二章 （章节名）（t=起止时间）

（逐章重复以上结构。）

## 关键概念

| 概念 | 解释 | 首次出现时间 |
| ---- | ---- | ------------ |
| （术语） | （按讲师语境用一两句话解释） | t=03:22 |
| … | … | … |

## 亮点与金句

- t=15:30 「（按语境修正后的引用原文）」——（可选：一句点评）
- …（5～10 条，在全片范围内分布，不要只集中在开头结尾。）

## 总结与启发

（3～5 句：全片价值、适合谁看、可迁移的方法或引发的思考。）
```
